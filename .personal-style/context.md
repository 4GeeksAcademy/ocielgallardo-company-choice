# Contexto Maestro del Proyecto

## Estado de partida

La web ya tiene una base visual funcional y profesional, pero se percibe rígida en algunos bloques por falta de recursos visuales y movimiento.

## Objetivo global

Mejorar la experiencia visual con imágenes y animaciones sin cambiar la dirección visual principal.

## Regla obligatoria para todo el proyecto

Toda decisión debe respetar el sistema de marca y diseño de HealthCore:

1. `.agents/rules/design.md`
2. `stitch_healthcore_design_system/healthcore_design_system_design.md`
3. `stitch_healthcore_design_system/clinical_intelligence_system/DESIGN.md`

## Alcance

Incluido:

1. Definir y seleccionar imágenes por sección.
2. Diseñar e implementar animaciones por sección.
3. Definir límites de rendimiento y accesibilidad.
4. Implementar por fases, iniciando con vertical slice en Hero.

No incluido:

1. Cambiar identidad de marca.
2. Romper patrón visual ya adoptado.
3. Introducir motion excesivo o decorativo.

## Método de ejecución

1. Avance lineal por GALs.
2. Cada GAL tiene objetivo, acciones, entregables y criterio de salida.
3. No se inicia el siguiente GAL sin cerrar el actual.

## Prioridad estratégica

1. Primero definir.
2. Luego implementar una referencia mínima validable.
3. Después escalar el patrón al resto de secciones.
4. Finalizar con revisión completa de calidad.

## Resultado esperado

Una web con mayor vida visual, más humana y más dinámica, manteniendo consistencia, claridad clínica y cumplimiento de marca.
