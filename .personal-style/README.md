# Personal Style Workspace - HealthCore

## Propósito

Este espacio documenta el trabajo visual del proyecto para avanzar con calma, por fases y con trazabilidad.

Objetivos principales:

1. Integrar imágenes con intención de marca.
2. Integrar animaciones útiles y sutiles.
3. Mantener consistencia visual en nuevas sesiones.

## Regla principal del proyecto

Siempre seguir las reglas de diseño y marca de HealthCore. Esta regla es obligatoria en todas las fases y sesiones.

Fuentes oficiales:

1. `.agents/rules/design.md`
2. `stitch_healthcore_design_system/healthcore_design_system_design.md`
3. `stitch_healthcore_design_system/clinical_intelligence_system/DESIGN.md`

Ningún cambio visual puede contradecir esas fuentes.

## Cómo se va a trabajar

1. Ejecución lineal por GALs.
2. No saltar fases.
3. Cerrar decisiones antes de implementar.
4. Validar mobile y desktop en cada GAL.
5. Registrar todo en `.personal-style/contexts.md`.

## Estructura de este directorio

1. `README.md`: guía general de trabajo.
2. `context.md`: contexto maestro del proyecto y visión de ejecución.
3. `GALs.md`: especificación completa de fases GAL.
4. `contexts.md`: bitácora viva de sesiones.

## Reglas de calidad continuas

1. Claridad por encima de decoración.
2. Movimiento sutil y con propósito.
3. Imágenes reales, limpias y profesionales.
4. Accesibilidad y legibilidad primero.
5. Coherencia visual entre secciones.

## Checklist al iniciar una sesión

1. Leer este `README.md`.
2. Leer `context.md`.
3. Leer `GALs.md`.
4. Revisar último registro en `contexts.md`.
5. Confirmar cumplimiento de reglas de marca.

## Checklist al cerrar una sesión

1. Registrar decisiones.
2. Registrar cambios.
3. Registrar riesgos y pendientes.
4. Dejar definido el próximo paso exacto.
