# Rule: Documentation

## Purpose
Standardize AI-generated documentation quality and scope.

## Requirements
- Write documentation in professional English.
- Keep content concise, factual, and easy to scan for AI agents.
- Use `TODO:` markers for unknown or missing information.
- Do not invent business requirements not present in `CONTEXT.md`.
- Avoid duplication across rule documents; each rule file owns one topic.

## Update Policy
- Update `memory-bank/progress.md` when major docs or governance context changes.
- Keep `memory-bank/projectbrief.md` focused on business mission and scope.
- Keep `memory-bank/techContext.md` focused on stack, structure, and conventions.

## Quality Bar
- Prefer short sections and clear headings.
- Avoid speculative language.
- Reference concrete repository artifacts when possible.
